Excel Data Load Error Debugger

Fix server 500 errors when loading Excel data from OneDrive

Error: Server 500 Internal Error

Your server is returning a 500 status code, which indicates an internal server error when trying to load Excel data.

GET https://deviceviewer.onrender.com/api/excel-data?folderName=__Oakridges%20Surgicentre%2FOakridge%20Surgi%20Centre%202024%20July 500 (Internal Server Error)

This error occurs when the server encounters an unexpected condition that prevents it from fulfilling the request.

Quick Fixes

The 500 error indicates a server-side issue. Here are the most likely solutions:

1. Check Server Logs

The 500 error is a generic server error. You need to check your server logs for more specific information:

  • Access your server console on Render.com
  • Look for error messages related to the Excel data loading
  • Check for authentication errors with Microsoft Graph API
  • Look for file path or permission issues
// Example of what to look for in server logs Error: Invalid authentication token Error: File not found Error: The specified item could not be found in the drive Error: Permission denied

2. Verify File Path and Naming

Your server is looking for an Excel file with a specific naming pattern:

// Server code constructs the filename like this: const parts = folderName.split('/'); const fileNameBase = parts.join('_'); const excelFileName = `${fileNameBase}_equipment_data.xlsx`;

For your folder path, the server is looking for:

__Oakridges_Surgicentre_Oakridge_Surgi_Centre_2024_July_equipment_data.xlsx

Make sure:

  • The Excel file exists with this exact name
  • The file is in the correct folder in OneDrive
  • There are no typos in the folder or file names

Use the button below to test if your server can connect to the Microsoft Graph API:

4. Verify Authentication

Your server uses the Microsoft Graph API to access OneDrive files. Ensure:

  • Your Azure AD credentials are correct (TENANT_ID, CLIENT_ID, CLIENT_SECRET)
  • Your Azure App has the necessary permissions (Files.ReadWrite.All)
  • Your access token is valid and not expired
  • The ONEDRIVE_USER has access to the files

Most Common Cause: Expired client secret or incorrect API permissions.

5. Debug Your Server Code

Add detailed error logging to your server to identify the exact issue:

// Enhanced error handling for /api/excel-data endpoint app.get('/api/excel-data', requireLogin, async (req, res) => { const { folderName } = req.query; console.log('Fetching Excel data for folder:', folderName); try { const token = await getAccessToken(); console.log('Access token acquired successfully'); const parts = folderName.split('/'); const fileNameBase = parts.join('_'); const excelFileName = `${fileNameBase}_equipment_data.xlsx`; console.log('Looking for Excel file:', excelFileName); const fileUrl = `${GRAPH_ROOT}/users/${ONEDRIVE_USER}/drive/root:/${ONEDRIVE_FOLDER_PATH}/${folderName}/${excelFileName}:/content`; console.log('OneDrive file URL:', fileUrl); const response = await axios.get(fileUrl, { headers: { Authorization: `Bearer ${token}` }, responseType: 'arraybuffer' }); console.log('Excel file downloaded successfully'); // Process the Excel file... } catch (err) { console.error('Detailed Excel fetch error:', { message: err.message, stack: err.stack, response: err.response?.data }); res.status(500).json({ error: 'Could not fetch Excel file', details: err.message }); } });

6. Check File Existence

Before trying to download the Excel file, check if it exists:

// Add this to your /api/excel-data endpoint // After getting the token, check if file exists const fileCheckUrl = `${GRAPH_ROOT}/users/${ONEDRIVE_USER}/drive/root:/${ONEDRIVE_FOLDER_PATH}/${folderName}/${excelFileName}`; try { const fileCheck = await axios.get(fileCheckUrl, { headers: { Authorization: `Bearer ${token}` } }); console.log('File found:', fileCheck.data); } catch (err) { console.error('File not found.
